WEB & APP DEV.

  • Each team can have 2-4 participants.
  • Any programming language, framework, or tool is allowed.
  • The project should be a functional web or mobile application relevant to the given problem statement, the problem statement will be given on spot.
  • Participants must submit their works within the alloted time.
  • Submissions must be original, Open-source libraries and APIs are allowed but must be credited.
  • Teams must present their work in a short demo (5-10 minutes), explaining key features and technologies used.
  • Projects will be evaluated based on creativity, functionality, user experience, and technical implementations.
  • Applications must not contain harmful, unethical, or offensive content.
  • Internet access will not be provided and participants should bring their own devices.
